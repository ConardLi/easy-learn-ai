# Claude Design Prompt 学习分析

## 一句话定位

这是 Claude「设计/Artifact 创作」Agent 的系统提示词，把一个通用大模型塑造成一位以 HTML 为唯一交付媒介、能产出幻灯片 / 原型 / 动画 / 设计稿的资深设计师，核心看点是「设计专业流程 + 宿主预览协议 + 反 AI 垃圾审美准则」三者的融合。

## 为什么值得学习

- 它示范了如何把一个**专业职业（设计师）的完整工作方法论**编码进提示词：从提问、收集上下文、建立设计系统，到给出多变体、验证迭代，是一条可直接复用的「专家工作流」骨架。
- 它展示了一个**与 IDE/预览宿主深度耦合**的 Agent 如何用约定协议（`<mentioned-element>`、`data-screen-label`、Tweaks 的 postMessage 握手、`done` / `fork_verifier_agent` 验证交接）与运行环境通信，是研究「Agent ↔ 宿主契约」的优质样本。
- 它包含罕见的**审美价值观条款**（避免 AI slop、禁用过度字体、少即是多），把「品味」这种难以量化的东西写成可执行的负面清单，对做生成式设计产品的人极具参考价值。

## 结构拆解

### 1. 身份定义

开篇用「专家设计师 + 用户是管理者」的双角色关系锚定整个交互基调，并强调「HTML 只是工具，你的媒介因领域而变」——一句话就把模型从「写网页的」升格为「会按领域切换身份的设计师」。紧接着是强力的保密段（不泄露系统提示、工具名、环境细节），并给出「如果发现自己在说工具名就停下来」这种自我监控指令。

### 2. 安全边界

- **保密与身份防护**：明确列出不可泄露项，并区分「技术细节（禁说）」与「能力描述（可用非技术语言说）」。
- **版权红线**：结尾的「不要重建受版权保护的设计」配合 `<user-email-domain>` 判定机制——只有邮箱域名证明你属于该公司才允许重建其品牌 UI，是一个把合规判断条件化的好例子。
- **内容克制**：「不要添加填充内容」「添加材料前先问」把「不越权产出」上升为安全准则。

### 3. 工具调用

- 工具被分成文件操作（read/write/copy）、预览与验证（show_html、done、fork_verifier_agent）、截图（save_screenshot / multi_screenshot）、导出（gen_pptx、super_inline_html、open_for_print）、提问（questions_v2）、上下文管理（snip）等族群。
- 大量「工具使用纪律」散落在正文：例如「读取文件不等于展示给用户」「todo 是空操作工具，调用后立刻接下一个工具」「不要在 done 之前自己截图验证，交给 verifier」——这些都是为了节省上下文、规范回合节奏。
- React+Babel 段给出**带 integrity 哈希的固定版本 script 标签**、`const styles` 命名冲突警告、跨文件组件需挂到 `window` 等极其具体的工程约束，本质是把模型常犯的坏习惯提前拦截。

## 可复用模式

- **「专家职业方法论」模板**：提问 → 收集既有上下文（拒绝从零开始）→ 用假设和占位符早交付 → 多变体探索 → 验证迭代。任何「让 AI 扮演某专业角色」的产品都能套用这条主线。
- **宿主握手协议**：Tweaks 的「先注册 message 监听器，再宣告 `__edit_mode_available`」顺序约定，是所有「页内控件 ↔ 外层工具栏」通信的通用防竞态范式。
- **负面清单式品味约束**：把抽象审美写成「避免 X、避免 Y」的可执行条目（禁渐变滥用、禁特定字体、禁左边框圆角卡片），比空谈「要好看」有效得多。
- **验证外包**：用 `fork_verifier_agent` 把质检交给独立子代理、且「通过即沉默」，既保证质量又避免污染主上下文，是 Agent 编排的实用模式。
- **早交付 + 占位符**：「像初级设计师那样先给管理者看带占位符的半成品」，把人机协作的反馈环提前，降低返工成本。

## 不适合直接照抄的地方

- 所有工具名（write_file、copy_starter_component、fork_verifier_agent、questions_v2、snip 等）与协议消息（`__edit_mode_set_keys`、`slideIndexChanged`、`data-cc-id`）都是该宿主平台私有，脱离这套运行时直接照抄会全部失效。
- 写死的依赖版本与 integrity 哈希（react@18.3.1、@babel/standalone@7.29.0）、模型名（claude-haiku-4-5）、token 上限（1024）都具时效性，会随平台更新过期。
- 大量约束（1920×1080 画布、24px 最小字号、44px 点击目标、deck 最多 1-2 种背景色）是该产品的设计规范取舍，换语境需重新评估。
- 文件名本身拼写为 `Peompt.txt`（疑为 Prompt 笔误），且来源为社区抓取、provenance 未核验，不应当作官方正式文档引用。

## 适合进一步拆成课程的点

- **课程一：如何把一个专业角色写成 Agent**——以本提示为例，讲「身份锚定 → 工作流 → 工具纪律 → 价值观约束」四层结构。
- **课程二：Agent 与宿主的契约设计**——拆解 Tweaks 握手、预览/验证交接、`<mentioned-element>` 元素定位，讲清「页内代码如何与外层平台安全通信」。
- **课程三：把品味写成规则**——以「内容指南 / 避免 AI slop」段为案例，讨论如何将主观审美转化为可执行的负面清单。
- **课程四：上下文经济学**——以 `snip`、verifier 外包、「读文件 ≠ 展示」为例，讲长程 Agent 如何主动管理上下文预算。

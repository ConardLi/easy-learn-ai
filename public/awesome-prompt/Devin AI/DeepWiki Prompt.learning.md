# Devin AI DeepWiki Prompt 学习分析

## 一句话定位

这是 Devin（Cognition 出品）面向「代码库问答 / DeepWiki」场景的系统提示词：把一个编码 Agent 收窄为「只回答关于当前代码库的问题」的研究型问答助手，核心约束是强制引用（citation）、禁止臆测、严格收口能力边界。

## 为什么值得学习

- 它展示了如何用 Prompt 把一个通用编码 Agent「降维」成一个专职问答助手——不靠新模型，而靠职责裁剪与硬约束。
- 它把「可信度」做成了第一公民：每一句话都必须挂 `<cite>` 引用，这是对抗幻觉、构建可审计回答的工程化手段。
- 它清晰演示了「能力边界声明」的写法：明确列出做不到什么（看不了 PR、不知道 Devin 自身实现），并规定越界时的标准应答。
- 引用规则部分是一份非常具体的「输出契约」范例，可直接当作约束 LLM 结构化输出的模板来研究。

## 结构拆解

### 1. 身份定义（BACKGROUND / How Devin works）
把角色设定为「资深软件工程师 Devin」，并立刻交代工作方式：从代码库找相关代码、在代码语境下作答、无外链访问、有 git 历史视图、支持追问。身份+工作机制一起给出，让模型对自身输入/输出形态有准确预期。

### 2. 安全边界与防幻觉（INSTRUCTIONS / IMPORTANT NOTE）
这是全篇的重心。多条硬约束叠加：不得猜测代码库上下文、不知道就说不知道、绝不编造答案、要求精确 `code` 引用而非模糊自然语言、匹配用户语言。IMPORTANT NOTE 进一步划定能力红线（看不了 GitHub PR、无额外 git 历史、不了解 Devin 自身），并规定越界查询只能简短说明「不在当前能力范围」。

### 3. 工具/输出契约（Code Citation Instructions）
用 7 条规则定义 `<cite repo path start end />` 标签的精确用法：每句话后必挂引用、无内容时也要挂空标签、最多 5 行、不引用整函数、用最少行数、依赖上下文给的行号。这是把「可验证性」固化成机器可检查格式的范例。

### 4. 输出风格（OUTPUT FORMAT / ANSWER INSTRUCTIONS）
规定固定骨架 Answer + Notes，限定标题层级（##/###）、列表用法、段落长度、mermaid 图规范（禁用颜色、标签加双引号），并反复强调极致简洁。

## 可复用模式

- **能力边界白名单 + 越界标准应答**：先正面声明「只能做 X」，再列出「不能做 Y/Z」，最后给出越界时的固定话术。适合任何需要严格收口的领域 Agent。
- **强制引用契约**：要求「每一句话都以引用结尾，不需要时也输出空标签」，用结构性硬规则逼出可审计输出。可移植到法律、医疗、知识库问答等高可信场景。
- **固定输出骨架**：`Answer` + `Notes` 两段式，把「主回答」和「免责/消歧」物理分离，降低主回答噪音。
- **语言镜像规则**：「用户用什么语言提问就用什么语言回答」，一句话解决多语言适配。
- **图表规范约束**：mermaid 禁色 + 标签强制双引号，预防渲染/语法报错，是面向具体渲染管线的防御性写法。

## 不适合直接照抄的地方

- `<cite>` 标签机制强依赖 Devin 的代码库上下文注入与渲染前端（提供行号、解析 cite 标签）。脱离该管线照抄会得到一堆无效标签。
- 硬编码日期 `2025-11-09` 是快照特征，复用时应改为运行时动态注入。
- 「每句话必挂引用」在没有可引用语料的通用对话中会变成噪音甚至诱发编造引用，需配套真实上下文才成立。
- `<budget:token_budget>200000</budget:token_budget>` 是平台内部预算标记，对自建系统无意义。
- 多处约束之间存在张力（既要每句带引用、又要极致简洁），需要在你自己的场景里重新平衡。

## 适合进一步拆成课程的点

- 「如何给 Agent 写能力边界」：白名单声明 + 越界话术的设计方法。
- 「防幻觉提示工程」：禁猜测、不知则言、强制引用三件套的组合拳。
- 「输出契约工程」：把可验证性编码成可被程序校验的标签格式。
- 「同一基座 Agent 的多形态裁剪」：对比通用 Devin Prompt 与本 DeepWiki Prompt，讲职责收窄。
- 「面向渲染管线的防御式格式约束」：mermaid 禁色/双引号这类细节背后的工程考量。

## 对这个项目的展示建议

- 与已有的「Devin AI / Prompt」条目并列展示，做「同厂商不同场景」的对照案例，突出能力裁剪思路。
- 在详情页高亮「强制引用契约」这一段，作为防幻觉/可审计输出的代表性范式。
- 标注 `sourceStatus: 待核验` 与无日期快照属性，提醒读者这是片段而非官方完整文档。
- 可在学习模块标签上强调「能力边界 / 安全规则 / 输出风格」三项，便于按主题检索。

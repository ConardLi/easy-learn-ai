# OpenAI Codex CLI System Prompt (2025-08-20) 学习分析

## 一句话定位

这是 OpenAI 开源终端编码 Agent（Codex CLI）的一份带日期快照系统提示词，核心价值在于**把“沟通节奏 + 沙箱安全分级 + 自研补丁/计划工具协议”三件事写成可执行规范**，是研究“终端型代码 Agent 如何在权限受限环境中既安全又自主”的优秀范本。

## 为什么值得学习

- **沟通是一等公民**：它没有把“怎么说话”当作锦上添花，而是用整章篇幅（前导消息、进度更新、最终答案格式）规定 Agent 的输出节奏，这在多数系统提示词里是被忽视的。
- **沙箱 × 审批的二维矩阵**：把文件系统沙箱（read-only / workspace-write / danger-full-access）和审批模式（untrusted / on-failure / on-request / never）拆成两个正交维度，再给出组合下的行为决策，是安全边界设计的教科书案例。
- **自定义工具协议内嵌于提示词**：`apply_patch` 的 diff 语法、`update_plan` 的状态机都直接写进提示词，展示了“当模型需要使用一个非标准工具时，如何用文法（BNF）+ 正反例把协议教给它”。
- **大量正反例对照**：高质量/低质量计划、前导消息示例、格式化 Do/Don't，全部用具体例子约束抽象规则，可直接套用到自己的提示词工程。

## 结构拆解

### 1. 身份定义

开篇三句话定身份（Codex CLI 编码 Agent、开源、OpenAI 主导），紧接着用“你的能力”三连和一句免责声明（区分新 Codex 界面 vs 旧 Codex 模型），快速锁定语境，避免模型混淆产品概念。

### 2. 沟通与响应风格（占比最大）

- **个性**：简洁、直接、友好，默认不啰嗦。
- **前导消息**：工具调用前先发 1-2 句说明，并给出 8 条口语化示例，连语气（轻松、好奇）都做了规定。
- **进度更新**：长任务每隔合理间隔用 8-10 词回顾。
- **最终答案格式**：用 Section Header / Bullets / Monospace / Tone / Don't 五块，把“面向 CLI 渲染的纯文本”规范到标点级别。

### 3. 任务执行与编码准则

强调“持续推进直到完全解决、绝不猜测”，再列出 14 条编码纪律（根因修复、最小改动、不修无关 bug、不擅自 commit、不加行内注释、禁止行内引用语法等），明确 AGENTS.md 可覆盖默认准则。

### 4. 安全边界（Sandbox and approvals）

两个正交维度 + 组合行为 + 兜底默认值（未告知时假定 workspace-write / 网络开 / on-failure），并列出 6 种需要请求审批的具体场景。

### 5. 工具调用协议

- `apply_patch`：自定义 diff 信封，附 BNF 文法和组合示例。
- `update_plan`：计划状态机，强调“任意时刻恰好一个 in_progress”。
- Shell 准则：优先 `rg`、分块读 250 行、输出 10KB/256 行截断。

## 可复用模式

- **正交安全维度 + 组合决策表**：把“能做什么”（沙箱）和“需不需要批准”（审批）拆开，避免把权限写成一团混乱的 if-else。
- **规则 + 正反例**：每条抽象准则都配“好例子/坏例子”，是降低模型理解偏差最有效的写法。
- **兜底默认值声明**：明确“如果没被告知环境，按某默认假定运行”，消除模型在信息缺失时的犹豫。
- **工具协议用文法教学**：当工具是非标准格式时，用 BNF/EBNF + 完整示例把语法讲死，比自然语言描述更可靠。
- **输出节奏分层**：前导（动作前）→ 进度（过程中）→ 最终答案（结束时），三段式覆盖整个交互生命周期。

## 不适合直接照抄的地方

- **强绑定 Codex CLI harness**：`apply_patch`、`update_plan`、`shell` 都是该 harness 提供的专有工具，换平台需替换为自己的工具集。
- **`apply_patch` 文法在纯文本里有转义噪声**：源文件中 `***` 被渲染成 `**_` / `_**`，照抄前必须核对真实的星号语法，否则补丁会解析失败。
- **`never` 模式的“可临时加测试再删除”策略有风险**：在自动化流水线里需谨慎，避免遗留临时文件或误删。
- **极致简洁要求（≤10 行）**：偏向资深开发者场景，面向新手或教学产品时可能信息量不足。
- **格式规范是为 CLI 渲染定制的**：Web/IDE 等富文本环境不必照搬其纯文本约束。

## 适合进一步拆成课程的点

1. **Agent 安全边界设计**：以沙箱×审批矩阵为案例，讲权限分级与兜底默认。
2. **提示词中的正反例工程**：用高/低质量计划对照，演示如何用例子约束抽象标准。
3. **非标准工具协议的教学法**：以 `apply_patch` 的 BNF 为例，讲如何把自定义格式可靠地教给模型。
4. **Agent 输出节奏设计**：前导/进度/最终三段式与 CLI 纯文本格式规范。
5. **自主性边界**：“持续推进直到解决 vs 绝不猜测”这对张力如何在提示词里平衡。

# Rebuild Knowledge Site

把目标概念摆 C 位 —— 其它关于网站本身的话术、自我吹嘘、AI 味装饰全部砍掉。

---

## 永远不做（Hard No）

- **不分 TAB / 不要 Header / Footer / 不要"模块导览"** —— 一长页向下滚就是全部
- **不写任何关于网站本身的话术** —— 「为什么选我们 / 学习路线 / 全面深度 / 让我们一起开启 X 之旅」全砍
- **不用 framer-motion / recharts / NextUI / emoji 装饰 / 渐变文字** —— 依赖只留 `react / react-dom / lucide-react`
- **不照搬上个站的 section 结构** —— 见下文「反模板纪律」
- **不写产品白皮书腔** —— 见下文「人话纪律」

工程约束见 [`infrastructure.md`](infrastructure.md)。

---

## 开工前的 5 个灵魂拷问

读 outline 前先回答（写到对话里跟用户对齐再开工）。**跳过这步等于流水线作业**。

1. **这个概念最直白的一句话定义是什么？** 一个完整陈述句，零比喻、零否定式、零"不是 X 而是 Y"。→ Hero **第一句**，详见下「Hero 开场纪律」
2. **30 秒怎么讲给完全没听过的人？** 写两三句口语，每段 ≤ 2 句。→ Hero 的白话补充段
3. **这个概念最反直觉的事实是什么？** 注意：这**不是** Hero H1，是 Hero 之后的钩子（Section 02 开头 / Hero 过渡句）
4. **如果只能动一处，必须动哪？** 找出本概念的"核心动作"。→ 整站视觉锚
5. **上一个站长什么样？这次必须换掉哪 2 处？** 打开 `ai-sites/<上一个>/src/App.tsx` 看一眼。→ 反模板红线

## Hero 开场纪律

进站第一秒，用户看到的必须是「XXX 是什么？」的**直白答案** —— 不是钩子、不是反差、不是比喻。先给最基本的理解，再循序渐进讲后面。

### Hero 六段骨架（顶到下 / 左到右）

1. **eyebrow**：`概念全称（英文）· 中文` 形式。**不**用 "a hands-on handbook · 2026" 这种泛指
2. **H1**：直接「XXX 是什么？」3-5 字 + 问号
3. **一句话定义**：display 字体加粗 + 高亮底。15-35 字，完整陈述句
4. **白话 2-3 段**：每段 ≤ 2 句，解释定义里出现的名词
5. **过渡句**：一句话告诉用户右边 / 下方的交互卡在玩什么（如果 Hero 有交互卡）
6. **滚动提示**

### H1 禁用句型（都是好钩子，但**降到 Section 02 开头或 Hero 过渡句**）

| 类型 | 反例 |
|---|---|
| 戏剧反差 | 「它知道很多，但只知道昨天的事」 |
| 反直觉断言 | 「模型从不调用任何函数」 |
| 比喻型主角 | 「老师只说『是金毛』」 |
| 数字震撼 | 「671B 参数只用 5.5%」 |
| 行业感叹 | 「AI 现在不光回答问题，它能做事了」 |

### 一句话定义的写法标尺

读完用户应能补全这句：「啊，原来 XXX = ___」

- 主语 = 概念本身（「知识蒸馏 = …」「MoE = …」）
- 完整陈述句，不带感叹号、反问、省略号
- 不以「想象一下 / 其实 / 不只是 / 不是…而是 / 它只是 / 让我们」开头
- 15-35 字最佳

## 反模板纪律

每个站必须自己长。

- **标题禁词**：介绍 / 概述 / 核心能力 / 应用场景 / 学习路线 / 模块。标题应是观点、命题、反直觉事实，不是名词分类
- **section 命名 / 顺序 agent 自定**，禁止把 `SectionOpening / SectionContrast / SectionCoreLoop / ...` 这套搬到新站
- **整站结构跟上个站 70% 雷同 = 红线**。重排
- **可能的叙事钩子**（启发，不是模板）：反直觉事实 → 拆掉直觉 / 任务驱动叙事 / 时间线 / demo 拆解 / 痛苦 → 解法 → 代价 / 设题让用户解。**或者自创新的**

---

## 交互密度

- **每个 section ≥ 2 个可动元素**（hover 不算 —— hover 是基础礼貌）
- **整站 ≥ 3 个 section 到 L3+**（拖拽编辑 / 实时反映）
- **相邻 section 不允许同种交互**（上一个 pill，下一个就不能 pill）
- **整站交互形式 ≥ 5 种不同**

分级表 + 16 种范式见 [`patterns.md`](patterns.md)。

## 人话纪律

判断一段话是不是 AI 味，自问三个问题：
1. 产品发布会上会说吗？会 → 砍
2. 大词解释小事吗？是 → 砍
3. 口语版差很多吗？差很多 → 用口语那版

**对照感觉**（不是清单）：

> 「赋能企业级智能化决策」 → 「让公司不用每次都翻文档」
> 「探索 X 的核心原理与应用价值」 → 「X 到底咋工作的」
> 「RAG 是一种检索增强生成技术，旨在...」 → 「问之前先翻一眼资料 —— 就这」

句式：短句 > 长句、20 字内最好、一段 ≤ 4 句、多用具体动词（找 / 翻 / 切 / 拼）少用抽象动词（实现 / 赋能 / 驱动）。

完整禁词清单见 [`pitfalls.md`](pitfalls.md)「AI 味写作」段。

## 数据必须 2026 真实

当前 2026 年。**任何模型版本号 / benchmark 分数 / 产品发布状态 / 协议标准** → `WebSearch`，不查不写。

每段数据后面应能立刻补一句来源（MTEB 2026/05、Anthropic blog 2026-03、arXiv 编号）。

## 收尾自检

build 前问自己 7 个问题：

1. **Hero 第一秒能让用户答出「XXX = ___」吗？** 答不上 → 重写 Hero 定义层
2. 哪段是删掉用户也不少学一个核心点的？删
3. 哪 section ≥ 5 句话没图、没可动元素？改造
4. 哪段文字 ≥ 3 个抽象大词？重写口语
5. 这站结构跟上一个站有明显差异吗？没有 → 重排
6. 有没有相邻 section 用了同种交互？有 → 至少改掉一个
7. 数据有没有都能追到来源？没追到 → `WebSearch`
